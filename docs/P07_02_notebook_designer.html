Approche “Modèle sur mesure simple”¶

On utilise le Concepteur / Designer d’un service Azure Machine Learning, interface graphique drag & drop, pour développer rapidement un modèle classique permettant de prédire le sentiment associé à un tweet.

On compare deux modèles entraînés puis on sélectionne le meilleur pour un déploiement.

Avant d'entraîner nos modèles, on prépare nos données :

Text preprocessing¶

  • On échantillone notre jeu de données pour ne garder que 2% de tweets, équilibré sur la variable target
  • On nettoie les tweets :
    • Expand verb contractions
    • Remove stop words
    • Use lemmatization
    • Detect sentences by adding a sentence terminator "|||" that can be used by the n-gram features extractor module
    • Normalize case to lowercase
    • Remove numbers
    • Remove special characters
    • Remove duplicate characters
    • Remove email addresses
    • Remove URLs
    • Normalize backslashes to slashes
    • Split tokens on special characters
  • On supprime les colonnes inutiles

Text vectorization¶

On va vectoriser nos tweets de 2 manières pour nos 2 modèles :

Feature Hashing¶

  • Paramètres :
    • Hashing bitsize: 10
    • N-grams: 2

Extract N-Gram Features from Text¶

  • Paramètres :
    • N-grams size: 2
    • Weighting function: TF-IDF Weight
    • Minimum word length: 3
    • Maximum word length: 25
    • Minimum n-gram document absolute frequency: 5
    • Maximum n-gram document ratio: 1
    • Normalize n-gram feature vectors: True

Entraînement des modèles¶

On va entrainer une Régression logistique sur les 2 types de vectorisatiobs précédentes:

  • Paramètres:
    • Optimization tolerance: 1e-07
    • L2 regularization weight: 1.0

Résultats¶

Le modèle avec la vectorisation N-Gram Features est le plus performant, voici les métriques :

  • Précision : 0.725
  • ROC AUC : 0.81
