Approche “Modèle sur mesure simple”¶

On utilise le Concepteur / Designer dun service Azure Machine Learning, interface graphique drag & drop, pour dvelopper rapidement un modle classique permettant de prdire le sentiment associ un tweet.

On compare deux modles entrans puis on slectionne le meilleur pour un dploiement.

Avant d'entraner nos modles, on prpare nos donnes :

Text preprocessing¶

  • On chantillone notre jeu de donnes pour ne garder que 2% de tweets, quilibr sur la variable target
  • On nettoie les tweets :
    • Expand verb contractions
    • Remove stop words
    • Use lemmatization
    • Detect sentences by adding a sentence terminator "|||" that can be used by the n-gram features extractor module
    • Normalize case to lowercase
    • Remove numbers
    • Remove special characters
    • Remove duplicate characters
    • Remove email addresses
    • Remove URLs
    • Normalize backslashes to slashes
    • Split tokens on special characters
  • On supprime les colonnes inutiles

Text vectorization¶

On va vectoriser nos tweets de 2 manires pour nos 2 modles :

Feature Hashing¶

  • Paramtres :
    • Hashing bitsize: 10
    • N-grams: 2

Extract N-Gram Features from Text¶

  • Paramtres :
    • N-grams size: 2
    • Weighting function: TF-IDF Weight
    • Minimum word length: 3
    • Maximum word length: 25
    • Minimum n-gram document absolute frequency: 5
    • Maximum n-gram document ratio: 1
    • Normalize n-gram feature vectors: True

Entraînement des modèles¶

On va entrainer une Rgression logistique sur les 2 types de vectorisatiobs prcdentes:

  • Paramtres:
    • Optimization tolerance: 1e-07
    • L2 regularization weight: 1.0

Résultats¶

Le modle avec la vectorisation N-Gram Features est le plus performant, voici les mtriques :

  • Prcision : 0.725
  • ROC AUC : 0.81
